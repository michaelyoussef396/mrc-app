### BUG-004 — Call button on lead detail
- **Surface:** `/admin/leads/<id>` — Call button (multiple instances)
- **Repro:** Click Call on desktop with no `tel:` handler → behaviour ambiguous; works if handler is registered, no-ops otherwise
- **Wave:** 2
- **Status:** pending fix (anchor + copy-to-clipboard fallback)

**Claude Code fix notes:**
- Root cause: `handleCall` set `window.location.href = "tel:..."` programmatically. With no OS `tel:` handler the browser could land on `about:blank` or behave inconsistently across browsers.
- Fix: removed `handleCall` JS handler. Converted all 5 Call button instances (status-specific CTAs, desktop header, mobile icon, mobile dropdown item) to `<Button asChild><a href={`tel:${lead.phone}`}>` pattern. Added a Copy icon button in the Contact Information card's phone renderReadOnly using `navigator.clipboard.writeText()` + toast.success("Phone copied").
- Files changed: `src/pages/LeadDetail.tsx`
- Commit: `eeaec8f`
- Verified by: Run 2 ✅ — confirmed on mrcsystem.com: clicking Call no longer causes about:blank, page stays put. Copy phone affordance: toast + clipboard ✅
